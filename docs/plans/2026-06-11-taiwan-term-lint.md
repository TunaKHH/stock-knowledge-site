# 待辦:防止中國用語混入文章 — 自動化檢查

> 建立日期:2026-06-11 · 狀態:**待實作**

## 背景

文章 `src/content/docs/基本面與技術面/什麼是基本面與技術面.md` 出現中國用語
`管理層質量`(台灣「質量」=物理 mass),已手動改為 `經營團隊素質`。
本站讀者為台灣投資人,需要一個**不依賴 Claude 記憶、自動擋下來**的機制,避免再發生。

## 決策

- **不用 skill** — skill 靠 Claude 判斷該不該叫它才生效,跟這次失誤同源;且手動在
  編輯器寫的內容完全不經過它。
- **主防線:git pre-commit hook** — commit 前 grep `src/content/`,命中中國用語就
  印出檔名行號並擋下 commit。工具無關,不管 Claude / 手 key / 其他工具改都會過這關。
  設計成**警告 + 擋 commit,可用 `git commit --no-verify` 強制略過**(把判斷權留給人,
  不靜默改字)。
- **配套:CLAUDE.md 一句規則** — 「本站讀者為台灣投資人,一律繁中台灣用語,禁用中國
  用語」,讓 Claude 第一次就寫對,hook 只是保險。

## 詞彙庫來源:tongwen-dict

採用 [tongwentang/tongwen-dict](https://github.com/tongwentang/tongwen-dict)(同文堂字典)
而非手刻黑名單。調查結果:

- 相關對照表在 `src/word/*.s2t.ts`,格式是 JS 物件 `{ '中國詞': '台灣詞' }`。
  例:`it.s2t.ts` → `信息: '資訊'`、`屏幕: '螢幕'`;`general.s2t.ts` → `渠道: '管道'`。
- 雖然檔名是 `s2t`(簡→繁),但**很多 entry 是繁→繁的在地化對照**(信息、屏幕、渠道
  在簡繁同形),正是我們要的「中國繁體詞 → 台灣繁體詞」。
- **字元級 `s2t-char.json` 用不到**(文章已是繁體,問題在詞彙選擇不是簡繁轉換)。
- 用法:取對照表的**鍵(中國詞)當偵測黑名單,值(台灣詞)當建議替換詞**。

### 注意 / 待解

- **誤判**:有些詞兩岸通用或看語境(如物理的「質量」是合法的、「數據」台灣也用)。
  直接全量套用會誤報 → 需要從 tongwen 表**篩出一份精選清單**,或維護一份白名單例外。
- 詞庫存成獨立檔(如 `.taiwan-terms.txt` 或直接引 tongwen 的 json),方便日後加詞。
- 決定 hook 怎麼引 tongwen:是 build 時把對照表抽成純文字清單,還是 runtime 讀其
  json?(避免把整個 tongwen 套件當 runtime 相依)

## 下一步

1. 從 tongwen `src/word/*.s2t.ts` 抽出繁→繁在地化對照,篩成精選黑名單 + 建議替換。
2. 寫 pre-commit hook(grep `src/content/`,命中印出 `檔案:行 中國詞 → 建議詞`,exit 1)。
3. 在專案根目錄建 `CLAUDE.md`,加入台灣用語規則 + 連結本檔。
4. (可選)同一支檢查腳本接進 CI,PR 階段再擋一次。
