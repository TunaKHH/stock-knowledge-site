# 對話體文章風格指南

stock-knowledge-site 新文章的預設寫法:**一場師徒對話 + 一條生活比喻**,把一個窄主題講到散戶也懂。

## 角色卡(全站固定,不要換名字)

| 角色 | 設定 | 在對話裡的功能 |
| :-- | :-- | :-- |
| **實習生** | 投資新手、好奇、白話。偶爾自嘲(「我會不會像奧客一樣…」)。 | 提出讀者心裡的直覺疑問、把抽象往生活拉、用自己的話覆述(費曼確認)。 |
| **貝肯** | 資深導師,耐心、會打比方,不掉書袋。 | 用比喻錨定 → 對應到主題 → 收斂出重點。糾正誤解。 |

對話以「實習生問 / 貝肯答」交替推進。需要連續說明時用敘述句帶,例如「貝肯接著解釋:」。

## 結構模板

固定六拍(對應 `assets/article-template.md`):

1. **破題**(🟧)— 讀者最直覺的疑問。貝肯先丟比喻,不急著定義。
2. **為什麼存在**(🟩)— 這東西補上什麼落差 / 解決什麼問題。實習生覆述一次。
3. **本體在講什麼 / 怎麼運作**(🟧)— 具體內容,維持比喻。
4. **常見誤解**(🟩)— 一個會搞錯的直覺,貝肯糾正。
5. **實務該抓什麼**(🟧)— 3–4 個可操作重點。
6. **簡單說**(`---` 後)— 用比喻一句總結 + 一句收尾。

節數可依主題增減,但「破題比喻 → 為什麼存在 → 本體 → 實務 → 簡單說」這條主線要在。

## 格式規則

- **小標**:`### 🟧 …` 與 `### 🟩 …` **交替**使用(橘、綠輪流),小標本身是一個問題或主題。
- **比喻要一條到底**:選一個生活場景(成績單/家長會、餐廳、球賽、租屋…),整篇用它的詞彙(老師=管理層、家長=投資人…)。**一篇只用一個主比喻**,別中途換。
- **少表格、少術語牆**:對話體靠口語推進。最多 1–2 個短列表(像「去哪裡聽」),不要堆規格表——那是舊說明體的作法。
- **術語第一次出現**用粗體 + 括號原文,例如 **展望(guidance)**、**資本支出(capex)**。
- **收尾**用 `簡單說:**<比喻總結>。**` 一句,呼應開頭比喻。

## 比喻挑選

挑「讀者百分百有經驗」的日常場景,且各角色能一一對應到主題的組成。已用過的別重複:

- 法說會 → **成績單(財報)vs. 家長會(法說會)**

挑比喻時自問:主題的每個關鍵組成,在這個比喻裡都有對應物嗎?有 → 好比喻;只對到一半 → 換。

## 紅線

- **禁 AI slop**(見專案 `DESIGN.md`):不要漸層、彩色圖示圓圈、置中一切、過度熱情的語氣。對話要自然,別「演」——一來一往精簡,別為了像真人而塞廢話。
- **事實要查證**:涉及制度 / 數字 / 規範(如 Reg FD 公平揭露、公開資訊觀測站、ADR 換股比例)必須正確。不確定就查,別憑印象寫。
- **非投資建議**:解釋觀念,不喊進出、不報明牌。
